## 功能

| 功能 | 说明 |
|------|------|
| 📡 一键同步 | 添加好友/群聊，一键完成 QCE 导出→ChatLab 导入 |
| ⏰ 定时自动同步 | 每天/每周/每月，精确到分钟，到点自动执行 |
| 🤖 智能服务管理 | 自动检测 QCE/ChatLab 是否运行，未启动自动拉起 |
| 🪟 QQ 进程处理 | 同步时检测 QQ 运行状态，不打断用户 |
| 📜 同步记录 & 重试 | 区分手动/定时同步，失败可一键重试 |
| 🚀 一键/后台启动 | CMD 一闪即关，支持创建桌面快捷方式 |

## 技术架构

```
QCE（QQ Chat Exporter）
    ↓ 导出 JSON 文件到本地磁盘
QCE2ChatLab（Python FastAPI，端口 15520）
    ↓ Push API 导入
ChatLab（本地知识库，端口 3110）
    ← 浏览器控制页 http://127.0.0.1:15520
```

所有数据不出你的电脑。

## 完整安装指南

### 第一步：安装运行环境

1. 下载安装 **Python 3.11 或更高版本**
   - 访问 https://www.python.org/downloads/
   - 点击下载 Windows installer (64-bit)
   - 运行安装程序，**务必勾选 "Add Python to PATH"**
   - 安装完成后，打开 CMD 输入 `python --version` 确认显示版本号

2. 下载安装 **QCE（QQ Chat Exporter）**
   - 访问 https://github.com/shuakami/qq-chat-exporter/releases
   - 下载最新版本
   - 解压到任意目录（建议放桌面）
   - 首次启动 QCE，扫码登录 QQ
   - 确认 QCE 能正常导出聊天记录

3. 下载安装 **ChatLab**
   - 访问 https://chatlab.fun/cn
   - 下载最新 Windows 版本
   - 解压到任意目录
   - 首次启动，初始化知识库

### 第二步：下载 QCE2ChatLab

**方式一：从 GitHub 下载（推荐）**
```
1. 打开 https://github.com/Ruoan-486/QCE2Chatlab
2. 点击绿色的 "Code" 按钮
3. 选择 "Download ZIP"
4. 解压到桌面
```

**方式二：从门户网站下载**
```
1. 打开 https://ruoan486.icu
2. 找到 QCE2ChatLab 卡片
3. 点击 "⬇ 下载正式版" 或 "⬇ 下载开发版"
```

### 第三步：安装 Python 依赖

打开 CMD，进入解压后的 QCE2ChatLab 文件夹：

```cmd
cd C:\Users\你的用户名\Desktop\QCE2ChatLab
pip install fastapi uvicorn httpx pydantic
```

等待安装完成（可能需要几分钟）。

### 第四步：配置

**准备工作：确认以下三项工具都已启动：**
- ☑️ QCE 已经启动并正常登录 QQ
- ☑️ ChatLab 已经启动
- ☑️ Python 已经安装

**首次启动配置：**

1. 双击运行 **启动.bat**，等待终端显示类似以下信息：
   ```
   QCE2ChatLab v1.4.0
   INFO: Uvicorn running on http://127.0.0.1:15520
   ```

2. 打开浏览器，访问 `http://127.0.0.1:15520`

3. 进入 **⚙️ 设置** 页，配置以下内容：

   **添加同步会话：**
   - 输入好友 QQ 号或群号
   - 输入备注名称（方便识别）
   - 点击"添加"
   - 重复操作，添加所有要同步的会话

   **配置 QCE 连接：**
   - QCE 地址：默认 `http://127.0.0.1:40653`（一般无需修改）
   - QCE 导出目录：检查路径是否正确（默认在 QCE 的用户目录下）

   **配置 ChatLab 连接：**
   - ChatLab 地址：默认 `http://127.0.0.1:3110`（一般无需修改）
   - Token：从 ChatLab 设置页复制

   **配置定时同步（可选）：**
   - 开启"定时自动同步"
   - 选择类型：每天 / 每周 / 每月
   - 设置执行时间
   - 配置服务自启动脚本路径（让 QCE2ChatLab 自动维护 QCE/ChatLab）

### 第五步：运行

**方式一：一键启动（推荐，需要 QCE 和 ChatLab 快捷方式在桌面）**
```
双击 一键启动.bat
→ CMD 窗口一闪即关
→ 4 秒后浏览器自动打开 http://127.0.0.1:15520
```

**方式二：后台启动（静默无感）**
```
双击 后台启动.bat
→ 完全静默，无任何窗口
→ 手动打开浏览器访问 http://127.0.0.1:15520
```

**方式三：普通启动（带控制台窗口）**
```
双击 启动.bat
→ 显示控制台窗口（可查看实时日志）
→ 不要关闭此窗口，关闭即退出
→ 浏览器访问 http://127.0.0.1:15520
```

### 第六步：首次同步

1. 在浏览器控制页点击 **📡 同步** 标签
2. 点击 **▶ 开始同步**
3. 查看同步进度：
   - ✅ 成功：会话图标变绿
   - ❌ 失败：会话图标变红，可点击重试
4. 同步完成后，打开 ChatLab 查看导入的聊天记录

### 进阶：配置服务自启动

如果你希望 QCE2ChatLab 在定时同步时**自动维护 QCE 和 ChatLab 的运行状态**：

1. 进入 **💻 系统** 页
2. 在"服务自启动"区域：
   - QCE 启动脚本：填写启动 QCE 的快捷方式路径（如 `C:\Users\你的用户名\Desktop\QCE.lnk`）
   - QCE 端口：默认 40653
   - ChatLab 启动脚本：填写启动 ChatLab 的路径
   - ChatLab 端口：默认 3110
3. 开启"开机自启"（可选）

之后每次定时同步前，QCE2ChatLab 会自动检测并启动这些服务。

## 常见问题

### Q：启动后浏览器打不开？
检查端口 15520 是否被占用，修改 `启动.bat` 中的端口号。

### Q：同步失败显示"QCE 未运行"？
确保 QCE 已经启动并正常登录 QQ，然后在"服务自启动"中配置路径。

### Q：定时同步没有触发？
- 检查是否开启了"定时自动同步"
- 检查设置的执行时间是否正确
- 查看 📋 日志 页查找原因

### Q：同步时 QQ 弹窗处理？
- 如果 QCE 和 QQ 都正常运行，不会弹窗，直接静默同步
- 只有当 QCE 未运行且 QQ 在运行时，才会弹窗询问是否终止 QQ
